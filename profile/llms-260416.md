## Personal Experience (GPX)

GPX infers each visitor's real-time interests and reorganizes content automatically. No manual rules. No fixed segments. One web, a unique experience for every visitor.

### How GPX Works

#### Engine 1: Interest Signals & Real-Time Micro-Segmentation

- Captures clicks, scroll depth, programme comparisons, time on page, internal link clicks, and form interactions as interest signals
- Builds micro-segmentation profiles from the very first visit, with no prior historical data
- Does not rely on predefined groups or team-configured rules — the system infers them automatically
- Example: A visitor comparing three Health master's programmes and checking scholarships is a different profile from one who only visits rankings

#### Engine 2: Interest CDP + Dynamic Modules

- Tracks every signal and builds a real-time interest profile for each anonymous visitor
- Content blocks reorder based on each visitor's inferred interests — same page, different experience
- Example: Three visitors landing on Biotechnology degree page see completely different "You might also like" blocks:
  - **Undecided student** sees: student clubs, sports facilities, alumni testimonials
  - **Researcher** sees: research groups, recent publications, doctoral programmes
  - **Parent** sees: employability data, financial aid, campus housing

#### Engine 3: CRM Intelligence

- When a visitor identifies themselves (form), GPX sends a full interest profile to the CRM
- Instead of "Lead: John Smith," the admissions team receives: "John Smith. High interest in Health Sciences (70%) and Volunteering (30%). Visited the Medicine page 4 times in the last week."
- The first call stops being a cold introduction — it becomes an informed advising conversation
- Works with HubSpot, Salesforce, and other CRM platforms
- Profiles are generated automatically — no manual tagging or segmentation rules needed
- UTM parameters from campaigns can be used as initial interest signals
- Aggregated browsing patterns identify latent demand in academic areas not yet covered by existing programmes

### GPX vs. Traditional Personalization

| Traditional | GPX |
|---|---|
| Static segmentation: labelled "Alumni," only see alumni content | Fluid hyperpersonalisation: sees what interests the visitor right now |
| Fixed content: "This block always shows X" | Relevance guarantee: shows what matters most for this specific visitor |
| Complex manual rules: someone must configure and maintain every segment | Signal creativity: decide which actions matter and how much they weigh |

## Technology & Architecture

### MACH Architecture

Griddo is structured according to MACH design principles:

- **Microservices** — Modular components for flexibility. Each module has its own schema, styles, and logic.
- **API-First** — Decoupled architecture over flexible APIs, secured with JWT. GraphQL and REST API. Separate Private API and Public API.
- **Cloud-Native** — Serverless deployment for enhanced security and reliability. Reduced maintenance costs, increased scalability, edge performance.
- **Headless** — Complete separation between back-end and front-end. Griddo exposes all content through APIs, allowing any technology (Astro, Next.js, mobile app, voice assistant) to consume and display content.

### Technical Stack

- **Backend:** Node.js, TypeScript
- **Frontend rendering:** React, Next.js, Astro (marketing site)
- **Infrastructure:** AWS (serverless, Edge computing, Amplify)
- **CMS Engine:** Proprietary Griddo headless CMS
- **Search:** AI-powered semantic search engine

### Performance

- Static HTML generation at build time, distributed through global CDN
- Pages scoring **95+ on Core Web Vitals** on average
- LCP under 2.5 seconds
- Lighthouse scores above 95
- **Atomic Render** — Publish only changed pages; unchanged content is ignored
- **Parallel Site Queues** — Zero blocking between multiple site deployments
- **CheckHealth** — Smart scan before deployment to catch missing essentials
- Auto-scaling through CDN handles millions of visits per minute, even during admission peaks

### Scalability

- Can handle the traffic volumes that AWS infrastructure allows — millions of visits per minute
- Handles high web traffic during admission periods
- Automatic scaling based on demand
- Global CDN & Edge delivery — Distribution across 300+ edge nodes worldwide
- Minimal latency for students on any continent
